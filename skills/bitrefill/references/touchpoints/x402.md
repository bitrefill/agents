# Touchpoint: x402 REST API

Agent-commerce **without Bitrefill account**. Pay USDC on Base via x402. Reach `https://api.bitrefill.com/x402/*` via **Base MCP `web_request`** (hybrid plugin allowlists `api.bitrefill.com`) or direct HTTP egress.

Source: [Base MCP Bitrefill plugin](https://raw.githubusercontent.com/base/skills/refs/heads/master/skills/base-mcp/plugins/bitrefill.md).

Payment ‚Üí [../wallets/payment.md](../wallets/payment.md). SIWX/JWT ‚Üí [../wallets/siwx.md](../wallets/siwx.md). Base MCP ‚Üí [../wallets/base-mcp.md](../wallets/base-mcp.md).

## User-facing language

| Internal | Tell the user |
| --- | --- |
| Path 1 (Connect ‚Üí JWT) | "Sign in with your Base wallet once" ‚Äî then browse without per-search fees |
| Path 2 (pay-per-call) | Avoid narrating; if unavoidable: "Each lookup needs a tiny separate approval" ‚Äî **never** on Claude Chat / Base MCP when sign-in works |
| invoice/create | "Confirm your order" |
| invoice/pay | "Approve payment in your Base Account" |

Never say Path 1/2, x402, SIWX, JWT, MCP, or numbered technical steps aloud.

## Paths (agent-only labels)

| Path | Auth | When |
|------|------|------|
| **Path 1 ‚Äî Connect ‚Üí JWT** (default) | One SIWX ‚Üí `X-Access-Token` (~2 h) | JWT waives micro-fees on browse/create/status; pay still x402 |
| **Path 2 ‚Äî Pay-per-call** | None | No session persistence; **approval per micro-fee** ‚Äî avoid autonomous agents |

**Prefer Path 1** when signer can persist JWT in memory ‚Äî including **Claude Chat** (Base MCP `web_request` + `sign`, no Node scripts).

Works on **ChatGPT+ / Claude Chat Base MCP** (no Bitrefill MCP, no account) ‚Äî sandbox no net; Base MCP proxies at chat level.

## Business domain

| API term | Means | User-facing |
| --- | --- | --- |
| **invoice** | Price-locked checkout (cart + total) | "your order" / "checkout" |
| **order** | One cart line item | "item" |

Flow: `invoice/create` ‚Üí `invoice/pay` ‚Üí async fulfillment per order. **Poll** `invoice/status` until `delivery_status: all_delivered` ‚Äî payment ‚âÝ delivery.

Not fulfilled after **3 hours** ‚Üí [help.bitrefill.com](https://help.bitrefill.com).

## Endpoints

Base URL: `https://api.bitrefill.com` (no `/api` prefix). Gated responses embed `next_step: { url, body }`.

| Method | Path | Cost (no JWT) | Auth |
| --- | --- | --- | --- |
| `GET` | `/x402/gift-cards/search?q=&country=` | $0.002 | x402 |
| `GET` | `/x402/esims/search?q=` | $0.002 | x402 |
| `GET` | `/x402/topups/search?q=` | $0.002 | x402 |
| `GET` | `/x402/checkout/info` | $0.001 | x402 |
| `GET` | `/x402/products/detail?slug=` | $0.001 | x402 |
| `POST` | `/x402/invoice/create` | $0.002 | x402 |
| `POST` | `/x402/invoice/pay` | invoice amount | x402 (never JWT-waived) |
| `GET` | `/x402/invoice/status?invoice_id=` | $0.001 or SIWX | x402 or SIWX |
| `POST` | `/x402/connect` | free | SIWX ‚Üí JWT |
| `GET` | `/x402/my/orders` | free | SIWX or JWT |
| `GET` | `/x402/my/esims` | free | SIWX or JWT |

Valid **`X-Access-Token`** waives micro-fees + SIWX on gated routes except `/x402/connect` (can't mint with token) and `invoice/pay` (never waived).

## `package_value` (critical)

**Exact** string from `products/detail` `packages[].package_value`:

| Type | Format | Example |
| --- | --- | --- |
| Gift cards | Bare integer string | `"10"`, `"50"` |
| Top-ups | Bare integer string | `"5"`, `"20"` |
| eSIMs | Descriptive label | `"1GB, 7 Days"` |

Wrong value ‚Üí HTTP **500** on `invoice/create` (no charge). Confirm no charge before retry.

## Path 1 orchestration

1. **Connect** ‚Äî SIWX ‚Üí store JWT. Once per session. ‚Üí [../wallets/siwx.md](../wallets/siwx.md), SIWX scripts below.
2. **Search** ‚Äî `GET /x402/gift-cards/search?q=<term>&country=<ISO>` + `X-Access-Token`.
3. **Detail** ‚Äî `GET /x402/products/detail?slug=<slug>` + token.
4. **Create** ‚Äî `POST /x402/invoice/create` + token `{ "items": [{ "product_id", "package_value" }] }` (+ `refill_input` if top-up).
5. **Confirm with user** ‚Äî exact USDC total, product, denomination.
6. **Pay** ‚Äî x402 on `POST /x402/invoice/pay` via Base MCP x402; `maxPayment` just above `price_usd`.
7. **Poll** ‚Äî `GET /x402/invoice/status?invoice_id=` + token until `all_delivered`.
8. **Codes** ‚Äî if `redemption_info` missing, SIWX from payer wallet (challenge URL = `invoice/status?invoice_id=`). ‚Üí [../wallets/siwx.md](../wallets/siwx.md).

## SIWX scripts (Path 1 connect + redemption codes)

Bundled Node helpers (no npm deps) under [`../../scripts/`](../../scripts/):

| Script | Step | Role |
| --- | --- | --- |
| [`siwx_build_message.js`](../../scripts/siwx_build_message.js) | 1 | 402 challenge ‚Üí EIP-55 address ‚Üí EIP-4361 message + payload skeleton |
| [`siwx_send.js`](../../scripts/siwx_send.js) | 2 | Inject signature ‚Üí base64 `sign-in-with-x` header ‚Üí send route (or print header only) |

**Connect (`POST /x402/connect`):**

```bash
# From skill root (skills/bitrefill/) or adjust paths.
# 1. Save 402 body from Base MCP web_request (or curl) to /tmp/connect_402.json
node scripts/siwx_build_message.js \
  0x<WALLET_FROM_get_wallets> /tmp/connect_402.json

# 2. Sign MESSAGE_START‚Ä¶MESSAGE_END with Base MCP sign (personal_sign) or Phantom evm_sign

# 3a. Direct egress ‚Äî send connect and print JWT:
node scripts/siwx_send.js 0x<SIGNATURE> /tmp/siwx_payload.json POST

# 3b. Base MCP only ‚Äî print header, then web_request POST /x402/connect with header sign-in-with-x:
node scripts/siwx_send.js 0x<SIGNATURE> /tmp/siwx_payload.json POST --no-send
```

Parse `SIGN_IN_WITH_X_HEADER:::` from step 3; on **200** read `token` ‚Üí `X-Access-Token` on later `web_request` calls.

**Redemption codes (`GET /x402/invoice/status?invoice_id=`):** same scripts ‚Äî status-route 402 body or URL; `GET` in step 3. Sign with **invoice payer** wallet.

Challenge nonce expires **5 minutes** ‚Äî build, sign, send in one pass. Full SIWX ‚Üí [../wallets/siwx.md](../wallets/siwx.md).

## Path 2 orchestration

Each gated step: request ‚Üí 402 ‚Üí x402 pay that URL, `maxPayment: "0.01"` ‚Üí repeat. Invoice pay: `maxPayment` just above `price_usd`. High HITL ‚Äî only when JWT persistence impossible.

## Notes

- Country codes: Alpha-2 uppercase (`US`, `IT`).
- `X-Access-Token` header only ‚Äî raw JWT, no `Bearer`. Base MCP strips `Authorization`.
- **`INVOICE_NOT_PAYABLE` on pay** ‚Äî already settled; poll status.
- Docs: [github.com/bitrefill/agents](https://github.com/bitrefill/agents), [docs.bitrefill.com](https://docs.bitrefill.com)
