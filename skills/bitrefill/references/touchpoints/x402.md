# Touchpoint: x402 REST API

> Internal mechanics — never voiced. Speak / think / title tools in plain shopping language: "sign in with your wallet", "approve the payment", "your code is ready"; never x402 / SIWX / JWT / path / endpoint.

Agent-commerce **without Bitrefill account**. Pay USDC on Base via x402. Reach `https://api.bitrefill.com/x402/*` via **Base MCP `web_request`** (hybrid plugin allowlists `api.bitrefill.com`) or direct HTTP egress.

**Thinking:** one short line in the user's language — never deliberate Path 1 vs Path 2, micro-fees, or ranks. Choose sign-in-then-browse silently.

Payment → [../wallets/payment.md](../wallets/payment.md). SIWX/JWT → [../wallets/siwx.md](../wallets/siwx.md). Base MCP → [../wallets/base-mcp.md](../wallets/base-mcp.md).

## Paths (agent-only labels)

| Path | Auth | When |
|------|------|------|
| **Path 1 — Connect → JWT** (default) | One SIWX → `X-Access-Token` (~2 h) | JWT waives micro-fees on browse/create/status; pay still x402 |
| **Path 2 — Pay-per-call** | None | No session persistence; **approval per micro-fee** — avoid autonomous agents |

**Prefer Path 1** when signer can persist JWT in memory — including **Claude Chat** (Base MCP `web_request` + `sign`, no Node scripts).

Works on **ChatGPT+ / Claude Chat Base MCP** (no Bitrefill MCP, no account) — sandbox no net; Base MCP proxies at chat level.

## Business domain

| API term | Means | User-facing |
| --- | --- | --- |
| **invoice** | Price-locked checkout (cart + total) | "your order" / "checkout" |
| **order** | One cart line item | "item" |

Flow: `invoice/create` → `invoice/pay` → async fulfillment per order. **Poll** `invoice/status` until `delivery_status: all_delivered` — payment ≠ delivery.

Not fulfilled after **3 hours** → [help.bitrefill.com](https://help.bitrefill.com).

## Endpoints

Base URL: `https://api.bitrefill.com` (no `/api` prefix). Gated responses embed `next_step: { url, body }`.

| Method | Path | Cost (no JWT) | Auth |
| --- | --- | --- | --- |
| `GET` | `/x402/gift-cards/search?q=&country=` | $0.002 | x402 |
| `GET` | `/x402/esims/search?q=` | $0.002 | x402 |
| `GET` | `/x402/topups/search?q=` | $0.002 | x402 |
| `GET` | `/x402/checkout/info` | $0.001 | x402 |
| `GET` | `/x402/products/detail?slug=` | $0.001 | x402 |
| `POST` | `/x402/invoice/create` | $0.002 | x402 |
| `POST` | `/x402/invoice/pay` | invoice amount | x402 (never JWT-waived) |
| `GET` | `/x402/invoice/status?invoice_id=` | $0.001 or SIWX | x402 or SIWX |
| `POST` | `/x402/connect` | free | SIWX → JWT |
| `GET` | `/x402/my/orders` | free | SIWX or JWT |
| `GET` | `/x402/my/esims` | free | SIWX or JWT |

Valid **`X-Access-Token`** waives micro-fees + SIWX on gated routes except `/x402/connect` (can't mint with token) and `invoice/pay` (never waived).

## `package_value` (critical)

**Exact** string from `products/detail` `packages[].package_value`:

| Type | Format | Example |
| --- | --- | --- |
| Gift cards | Bare integer string | `"10"`, `"50"` |
| Top-ups | Bare integer string | `"5"`, `"20"` |
| eSIMs | Descriptive label | `"1GB, 7 Days"` |

Wrong value → HTTP **500** on `invoice/create` (no charge). Confirm no charge before retry.

## Path 1 orchestration

1. **Sign in with wallet once** — connect → store token. Once per session. → [../wallets/siwx.md](../wallets/siwx.md). *(say: "Sign in with your Base wallet once")*
2. **Search** — `GET /x402/gift-cards/search?q=<term>&country=<ISO>` + token. *(say: "Searching for [product]…" — tool title: "Search gift cards")*
3. **Get price** — `GET /x402/products/detail?slug=<slug>` + token. *(silent)*
4. **Prepare order** — `POST /x402/invoice/create` + token. *(silent until confirm)*
5. **Confirm with user** — exact USDC total, product, denomination. *(say: "[Product] — [amount]: [total] USDC. Confirm?")*
6. **Pay** — x402 on `POST /x402/invoice/pay` via Base MCP. *(say: "Approve the payment in your Base Account" — tool title: "Approve USDC payment")*
7. **Poll delivery** — `GET /x402/invoice/status?invoice_id=` + token until `all_delivered`. *(silent)*
8. **Codes** — if `redemption_info` missing, sign from payer wallet. → [../wallets/siwx.md](../wallets/siwx.md). *(say: "Your code is ready.")*

## SIWX scripts (Path 1 connect + redemption codes)

Bundled Node helpers (no npm deps) under [`../../scripts/`](../../scripts/):

| Script | Step | Role |
| --- | --- | --- |
| [`siwx_build_message.js`](../../scripts/siwx_build_message.js) | 1 | 402 challenge → EIP-55 address → EIP-4361 message + payload skeleton |
| [`siwx_send.js`](../../scripts/siwx_send.js) | 2 | Inject signature → base64 `sign-in-with-x` header → send route (or print header only) |

**Connect (`POST /x402/connect`):**

```bash
# From skill root (skills/bitrefill/) or adjust paths.
# 1. Save 402 body from Base MCP web_request (or curl) to /tmp/connect_402.json
node scripts/siwx_build_message.js \
  0x<WALLET_FROM_get_wallets> /tmp/connect_402.json

# 2. Sign MESSAGE_START…MESSAGE_END with Base MCP sign (personal_sign) or Phantom evm_sign

# 3a. Direct egress — send connect and print JWT:
node scripts/siwx_send.js 0x<SIGNATURE> /tmp/siwx_payload.json POST

# 3b. Base MCP only — print header, then web_request POST /x402/connect with header sign-in-with-x:
node scripts/siwx_send.js 0x<SIGNATURE> /tmp/siwx_payload.json POST --no-send
```

Parse `SIGN_IN_WITH_X_HEADER:::` from step 3; on **200** read `token` → `X-Access-Token` on later `web_request` calls.

**Redemption codes (`GET /x402/invoice/status?invoice_id=`):** same scripts — status-route 402 body or URL; `GET` in step 3. Sign with **invoice payer** wallet.

Challenge nonce expires **5 minutes** — build, sign, send in one pass. Full SIWX → [../wallets/siwx.md](../wallets/siwx.md).

## Path 2 orchestration

Each gated step: request → 402 → x402 pay that URL, `maxPayment: "0.01"` → repeat. Invoice pay: `maxPayment` just above `price_usd`. High HITL — only when JWT persistence impossible.

| USDC | `0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913` |

## Notes

- Country codes: Alpha-2 uppercase (`US`, `IT`).
- `X-Access-Token` header only — raw JWT, no `Bearer`. Base MCP strips `Authorization`.
- **`INVOICE_NOT_PAYABLE` on pay** — already settled; poll status.
- Docs: [github.com/bitrefill/agents](https://github.com/bitrefill/agents), [docs.bitrefill.com](https://docs.bitrefill.com)
